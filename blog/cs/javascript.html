浮點數運算精度問題

JavaScript 的 Number 是 64 bit 的雙精度浮點數(IEEE754)。

加法的誤差

\(0.1+0.2 \neq 0.3\) 

console.log(0.1 + 0.2)

0.30000000000000004

減法的誤差

\(0.12 - 0.1 \neq 0.02\) 

console.log(0.12 - 0.1)

0.01999999999999999

乘法的誤差

\(3.21 \times 3 \neq 9.63\) 

console.log(3.21 * 3)

9.629999999999999

除法的誤差

\(0.3 \div 0.1 \neq 3\) 

console.log(0.3 / 0.1)

2.9999999999999996

三角函數的誤差

\(\sin(\pi) \neq 0\) 

console.log(Math.sin(Math.PI))

1.2246467991473532e-16

分配律有時不成立

\((0.1 + 0.2) \times 5 \neq 0.1 \times 5 + 0.2 \times 5\) 

var a = (0.1 + 0.2) * 5;
var b = 0.1 * 5 + 0.2 * 5;
console.log('(0.1 + 0.2) * 5 = ' + a);
console.log('0.1 * 5 + 0.2 * 5 = ' + b);

(0.1 + 0.2) * 5 = 1.5000000000000002
0.1 * 5 + 0.2 * 5 = 1.5

四捨五入誤差的影響

多跑了一次回圈，因為預期最後一次結果是3，結果得到1.9999999999999998。 

var start = 1;
var end = 2;
var x, h;

h = (end-start)/3
x = start;
console.log("x=",x);
while(x<end) {
  x=x+h;
  console.log("x=",x);
}

x= 1
x= 1.3333333333333333
x= 1.6666666666666665
x= 1.9999999999999998
x= 2.333333333333333

失去精確位數

太相近的數值相減的結果完全不可信

\(3.141592653589793\) - \(3.141592653585682 \neq 0.000000000004111\) 

var x = 3.141592653589793;
var y = 3.141592653585682;
var z = x - y;
console.log("x - y = ", z);

x - y =  4.11093381558203e-12
